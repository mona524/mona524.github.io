聊聊python PEP8

发表于 2017-11-24   |   分类于 python

PEP8主要是python代码的书写规范，指在提高代码的可读性。那就来看看。规范的python代码需要遵循哪写规则。

代码布局


缩进：每级缩进4个空格

括号中使用垂直隐式缩进或使用悬挂缩进。

# (垂直隐式缩进)对准左括号
foo = long_function_name(var_one, var_two,
    var_three, var_four)

# (悬挂缩进) 需多一层缩进
foo = long_function_name(
     var_one, var_two,
     var_three, var_four)

# (悬挂缩进) 但下面情况, 需再加多一层缩进， 和后续的语句块区分开来
def long_function_name(
     var_one, var_two,
     var_three,var_four):
     print(var_one)
my_list = [
    1, 2, 3,
    4, 5, 6,
    ]
result = some_function_that_takes_arguments(
    'a', 'b', 'c',
    'd', 'e', 'f',
    )

空格和tab:4个空格的缩进（编辑器都可以完成此功能），尽量不使用Tap，更不能混合使用Tap和空格。

最大行宽：每行最大行宽不超过 79 个字符，一般续行可使用反斜杠
括号内续行不需要使用反斜杠

# 无括号续行， 利用反斜杠
with open('/path/to/some/file/you/want/to/read') as file_1, \
     open('/path/to/some/file/being/written', 'w') as file_2:
 file_2.write(file_1.read())

# 括号内续行， 尽量在运算符后再续行
class Rectangle(Blob):

 def __init__(self, width, height,
     color='black', emphasis=None, highlight=0):
     if (width == 0 and height == 0 and
      color == 'red' and emphasis == 'strong' or
      highlight > 100):
        raise ValueError("sorry, you lose")
     if width == 0 and height == 0 and (color == 'red' or
          emphasis is None):
        raise ValueError("I don't think so -- values are %s, %s" %
        (width, height))

空行：两行空行用于分割顶层函数和类的定义
类中的方法前后用空行分隔
函数中的空行往往表示将代码进行逻辑分段

 # 类的方法定义用单个空行分割，两行空行分割顶层函数和类的定义。
class A(object):
    def method1():
        pass

    def method2():
       pass


def method3():
    pass

编码：python3中应该使用utf-8，python2中ascii

导入：1、导入的代码通常单独成行，尽量不要一起导入

#yes
import os
import sys
#允许同一个模块导入多个方法
from subprocess import Popen, PIPE 
#no 
import sys, os

2、导入的代码永远放在顶层，放在文件全局注释之下，并遵循以下规格：
（1）、标准模块的导入
（2）、第三方模块的导入
（3）、本地库
3、尽量绝对导入；from <module> import *这样的导入是不推荐的。

字符串：
单引号和双引号作用是一样的，但必须保证成对存在，不能夹杂使用. (建议句子使用双引号, 单词使用单引号, 但不强制.)

空格：
括号里边避免空格
逗号，冒号，分号之前避免空格
函数调用的左括号之前不能有空格
赋值等操作符前后不能因为对齐而添加多个空格
二元运算符两边放置一个空格
涉及 = 的复合操作符 ( += , -=等)
比较操作符 ( == , < , > , != , <> , <= , >= , in , not in , is , is not )
逻辑操作符( and , or , not )

注释：
注释块通常应用在代码前，并和代码有同样的缩进。每行以 ‘# ‘ 开头, 而且#后面有单个空格。
单行注释(应避免无谓的注释)

命名规则：
包和模块名应该简短，全部用小写字母, 多字母之间可以使用单下划线连接。
类名：遵循驼峰命名
全局变量名: 全局变量名应尽量只在模块内部使用, 对可能使用语句 from moduleName import variableName 而被导入的模块，应采用 all 机制来防止全局变量被别的模块导入, 或者在全局变量名开头加一个前置下划线._name = ‘name’
函数名：函数名应该为全部小写的凹驼峰规则
常量名 ：常量全部使用大写字母的凹驼峰规则来表示, 通常在模块顶格定义 MAX_OVERFLOW = ‘’

编程建议：
None 的比较用 is 或 is not，而不要用 ==，用 is not 代替 not … is, 前者的可读性更好

异常：异常类应该继承自Exception，而不是 BaseException；
捕获异常时尽量指明具体异常, 尽量不用 except Exception, 应该捕获 出了什么问题，而不是 问题发生

try/except 子句中的代码要尽可能的少, 以免屏蔽掉其他的错误

函数或者方法在没有返回值时要明确返回 None

使用 isinstance() 代替对象类型的比较

空序列类型对象的 bool 为 False:

不要用 == 进行 bool 比较
